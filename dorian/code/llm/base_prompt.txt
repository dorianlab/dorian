Just for your reference, here is the function used for matching the pattern of a RewriteRule with the graph:
def match(
    pattern: DAG, dag: DAG, comparator: Callable[[Node, Node], bool] = comparator
) -> Tuple[bool, Dict[UUID, UUID] | None]:
    """
    Match a rule pattern against a DAG and find valid candidate mappings.

    Args:
        rule_pattern: Pattern from the rule containing nodes and edges
        dag: The DAG to match against
        comparator: Function to compare nodes

    Returns:
        tuple: (bool indicating if match found, candidate mapping dict if found, else None)
    """

    # Gets one list and one item. returns the indices of the elements in the list, which were comparable/equivalent with the specified item.
    def _iter(elements, element, _comparator):
        for idx, el in elements:
            if _comparator(el, element):
                yield idx

    # Graph matching:
    # for each node in the rule, returns a list of potential matches, then iterates over all the possible mappings for different nodes.
    # If we cannot find for even a single node any matches, then we have failed to find the rule (product func returns empty)
    # Each element in product (values) is a list, long as the number of pattern nodes
    for values in product(
        *map(
            lambda x: _iter(dag.nodes.items(), x, comparator),
            pattern.nodes.values(),
        )
    ):
        # candidate nodes should have unique IDs
        # cannot match two nodes with a node
        if len(values) != len(set(values)):
            continue

        candidate = dict(zip(pattern.nodes.keys(), values))

        # Count matched edges
        matched = 0
        for edge in pattern.edges:
            s, d = candidate[edge.source], candidate[edge.destination]
            for _edge in dag.edges:
                if (_edge.source == s) & (_edge.destination == d):
                    matched += 1
                    break

        if matched == len(pattern.edges):
            # Meaning all edges are matched and we have a complete match.
            return True, candidate

    return False, None

class RewriteRule:
    pattern: Pattern

Lets look at the attributes of a RewriteRule:
pattern: is of type Pattern which is another name of DAG.
    class DAG:
        nodes: Dict[UUID, Node | Operator | Parameter] = field(default_factory=dict)
        edges: list[Edge] = field(default_factory=list)

    class Node:
        type: str = wildcard
        text: str = wildcard
    class Parameter: (This is the representative of a parameter of a function)
        name: str
        type: SupportedType
        value: str
        language: str
    class Operator: (This is the representative of a function)
        name: str
        language: str
    class Edge:
        source: UUID
        destination: UUID

transformations: Apply | Replace | Delete | Revert | ToOperator | ToParameter | Add
    class Add:
        nodes: list[Node] | None = None
        edges: list[Tuple[UUID, UUID]] | None = None

    class Apply:
        f: Callable[[DAG, Dict], DAG]

    class Replace:
        pass

    class Delete:
        nodes: list[UUID] = field(default_factory=list)

        nodes: list[UUID] = field(default_factory=list)

    class ToParameter:

And here is the _update function used in some of the available rules. You can also use it in your suggest rules.

def _update(dag: DAG, key: UUID, part: str, value: Any) -> DAG:
    # replaces a node with a new node, having the ID=key and the attribute part=value updated with the new value.
    if key not in dag.nodes:
        return dag
    node = Node(**dict(asdict(dag.nodes[key]), **{part: value}))
    return DAG(nodes=dict(dag.nodes, **{key: node}), edges=dag.edges)

As you can see, a rewrite rule can itself has some other rules in its "rules" attribute. You can call them the nested rules.
What is their point? When we want to apply a node, we look at its nested rules. If some exist, then we add those rules to the queue.
When new nested rules are added, these new rules are applied to all the subgraphs, although they may have been the product of the application of the original rule on a single subgraph.
Also, every time a rule is applied, all the inner rules are generated again, the only difference being, the mapping is new in these newly added nested rules.
